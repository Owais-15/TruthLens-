System Design

Neuro-Symbolic Architecture

Neural language understanding + Deterministic fact retrieval

📝

Stage 1 · Atomic Claim Extraction

Llama-3.1-8b (via Groq LPU) breaks input into isolated, falsifiable claims with character-level indices. Structured system prompting with JSON output.

↓
🔍

Stage 2 · Semantic Evidence Retrieval

Exa Neural Search API converts each claim to a dense-vector query and retrieves top-5 high-authority documents from a 10M+ document index.

↓
🧠

Stage 3 · Entailment Classification

Llama-3.1-8b acts as an NLI system, performing cross-attention between claim and evidence to classify: Entailment / Contradiction / Neutral.

Tech Stack

React + TypeScript Node.js + Express PostgreSQL + Drizzle ORM Groq LPU Inference Llama-3.1-8b-instant Exa Neural Search Redis Caching JWT Auth Tailwind CSS

Security & Trust

Robust Security Architecture

Multi-layer protection against technical and adversarial threats

🔐 Authentication

Stateless JWT authentication with 15-min access tokens + 7-day refresh tokens. Role-Based Access Control (Free / Premium / Enterprise) with monthly verification quotas.

🚦 Rate Limiting

Window-based algorithm strictly enforcing 100 requests / 15 minutes per user. Adaptive throttling for suspicious patterns.

🛡️ Adversarial Threat Mitigation

Evidence Poisoning: Requires ≥2 independent sources from different domains before marking a claim verified.
Prompt Injection: Strict JSON schema validation. User input sanitised before LLM calls.
DoS Attacks: Hard 5,000-character limit + cooldown periods for complex requests.

🔒 Privacy & Compliance

Ephemeral in-memory processing by default. AES-256 encryption at rest. TLS 1.3 in transit. GDPR-compliant with 24-hour deletion guarantee.

Infrastructure

Horizontal Scalability Design

Production-ready microservices with cost optimisation built in

☁️ Stateless Microservices

Node.js services containerised with Docker, orchestrated via Kubernetes. Auto-scales based on CPU/memory. PostgreSQL primary-replica with 3 read nodes.

⚡ Redis Caching Layer

Claims: 24-hour TTL (key: SHA-256 of input text)
Evidence: 1-hour TTL (key: SHA-256 of search query)
34% cache hit ratio reduces API costs significantly.

📊 Monitoring

OpenTelemetry distributed tracing across microservices. Auto-alerts when error rate >2% or latency >6s.

💰 API Cost Optimisation

1. Intelligent Claim Batching — Deduplicates similar claims, reducing API calls by grouping semantically equivalent queries.
2. Local Knowledge Base — 10,000 common facts (e.g., "Paris is the capital of France") verified locally without API calls.
3. Progressive Verification — Two-phase approach: fast Phase 1, thorough Phase 2 only if needed. Reduces average API calls by 40%.
